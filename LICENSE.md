The **Mm2 Script** supports a full configuration file that lets you personalize every module before launch. Save the following template as `config.lua` in the same directory as the main script file.

| Setting | Default Value | Description |
|---|---|---|

| `autoTradeEnabled` | `false` | Enables **Mm2 Script Auto Trade** module |
| `hitboxMultiplier` | `2.5` | Scale factor for **Murder Mystery 2 Script Hitbox** |
| `guiTheme` | `"dark"` | UI color scheme for the **Mm2 Script GUI** |

| `autoShootMode` | `"crosshair"` | Trigger mode for **Auto Shoot Mm2 Script** |

```bash
-- Mm2 Script Configuration File
-- Save as config.lua in your script root directory

local Config = {
    -- Visual Settings
    ESP = {
        Enabled = true,
        ShowNames = true,
        ShowDistance = true,
        RoleColors = {
            Murderer = Color3.fromRGB(255, 50, 50),
            Sheriff = Color3.fromRGB(50, 120, 255),
            Innocent = Color3.fromRGB(100, 255, 100)
        },
        MaxRenderDistance = 500
    },

    SilentAim = {
        Enabled = true,
        FOV = 120,
        ShowCircle = false,

        Enabled = true,
        Delay = 0.3,
        OnlyWhenMurderer = true
    },

    AutoShoot = {
        Enabled = false,
        Mode = "crosshair",
        TriggerDelay = 0.1
    },

    -- Economy Settings
    AutoTrade = {
        Enabled = false,
        MinimumValue = 50,
        AcceptOverpay = true
    },

    FreezeTrade = {
        Enabled = false,
        Duration = 30
    },

    -- Hitbox Settings
    Hitbox = {
        Enabled = false,
        Multiplier = 2.5,
        VisualIndicator = false
    },

    -- General
    GUI = {
        Theme = "dark",
        Keybind = Enum.KeyCode.RightShift,
        StartMinimized = false
    }
}

return Config
```

## 🛠️ Troubleshooting Common Issues

- ❌ **GUI Does Not Appear After Execution**
  The **Mm2 Script GUI** sometimes fails to render if the executor attaches before the game fully loads. 
  **Solution:** Wait until you are fully inside a Murder Mystery 2 lobby, then re-attach and execute the script again.

- ❌ **ESP Overlay Shows Wrong Roles**
  Occasionally the **Mm2 Script ESP** displays incorrect role colors during the first 2 seconds of a round. 
  **Solution:** This is a sync delay — roles update after the round intro animation finishes. No action needed.

- ❌ **Silent Aim Locks Onto Teammates**
  The **Mm2 Script Silent Aim** may target innocents when you are the Sheriff if `TargetPart` is set incorrectly. 
  **Solution:** Ensure `OnlyWhenMurderer` is `false` and verify your role filter settings in `config.lua`.

- ❌ **Auto Trade Accepts Bad Deals**
  The **Mm2 Script Auto Trade** module accepted a trade below your value threshold. 
  **Solution:** Update to v2.8.1 where the value-check logic was corrected, and set `MinimumValue` to at least `50`.

- ❌ **Script Crashes on Mobile Devices**
  The **Murder Mystery 2 Script** force-closes the executor on older Android devices.
